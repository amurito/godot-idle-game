## Prólogo: El Sistema como Organismo

Este no es un manual de economía tradicional. Es un tratado de ingeniería viva.

El sistema que vas a estudiar no es una máquina. Es un organismo: crece, se adapta, sufre estrés, metaboliza crisis, desarrolla memoria estructural y puede colapsar si sus ciclos vitales se rompen.

Cada variable no es un número. Es un órgano con función específica.

Este documento describe:
- la anatomía del sistema
- sus ciclos metabólicos
- sus puntos de colapso
- su capacidad de evolución

**Convención de lectura:**  
Las variables latinas (a, b, d, e) representan flujos de energía.  
Las variables griegas (μ, κ, ε, ω) representan estructura, cognición y entropía.  
Los subíndices (cₙ) indican estados dinámicos observados.  
Los superíndices (fⁿ) indican estados teóricos esperados.

## 1. El Organismo Económico

### 1.1 Definición del sistema vivo

El sistema económico evolutivo es una entidad que:
- **respira**: produce y consume energía (Δ$)
- **piensa**: organiza información (μ)
- **recuerda**: acumula estructura (cₙ, κμ)
- **sufre**: acumula distancia entre lo real y lo ideal (ε)
- **digiere**: metaboliza estrés mediante biosfera (biomasa, micelio)

No existe en equilibrio. Existe en tensión permanente entre:
- producción y agotamiento
- expansión y fricción
- complejidad y colapso

### 1.2 Ciclo vital completo

```
Energía (Δ$) → Cognición (μ) → Estructura (cₙ) → Estrés (ε) → Biosfera (B) → Estabilidad
```

Cada ciclo refuerza o degrada al siguiente.  
Un colapso en cualquier órgano afecta a todo el sistema.

### 1.3 Leyes fundamentales

**Primera ley termodinámica económica:**  
La energía del sistema se conserva en la forma Δ$ = Δ$_click + Δ$_pasivo

**Segunda ley termodinámica económica:**  
Todo crecimiento produce estrés (ε). Sin biosfera, el estrés acumulado colapsa el sistema.

## 2. Energía (Δ$)

### 2.1 Anatomía de la producción total

La energía del sistema se expresa como:

```
Δ$ = Δ$_click + Δ$_pasivo
```

Donde:

```
Δ$_click = clicks · (a · b · cₙ)
Δ$_pasivo = d · md · so + e · me
```

### 2.2 El músculo activo: click

**a — Click base**  
El impulso mecánico individual. Crece linealmente con cada mejora.  
*Qué mide:* esfuerzo directo no amplificado.  
*Cómo escala:* lineal, sin límite teórico.  
*Qué pasa si colapsa:* el sistema pierde su capacidad de intervención directa. Se vuelve completamente pasivo.

**b — Multiplicador de click**  
El amplificador exponencial suave del esfuerzo.  
*Qué mide:* tecnología aplicada al impulso manual.  
*Cómo escala:* exponencial contenido, evita explosiones descontroladas.  
*Qué pasa si colapsa:* el click se vuelve inútil frente a los productores pasivos.

**cₙ — Persistencia dinámica observada**  
El amplificador estructural. No es un número fijo: es la memoria del sistema en acción.  
*Qué mide:* cuánto del pasado estructural persiste en el presente.  
*Cómo escala:* converge hacia fⁿ mediante sigmoide.  
*Qué pasa si colapsa:* todo el esfuerzo acumulado se desvanece. Es amnesia económica.

### 2.3 El músculo pasivo: productores autónomos

**d — Trabajo Manual**  
Producción lineal y constante. No se detiene, no se acelera.  
*Qué mide:* capacidad de generar valor sin intervención activa.  
*Cómo escala:* lineal con cada unidad comprada.  
*Qué pasa si colapsa:* el sistema depende completamente del click. No hay autonomía.

**md — Ritmo de Trabajo**  
Multiplicador que acelera el trabajo manual sin cambiar su naturaleza.  
*Qué mide:* eficiencia organizacional aplicada a d.  
*Cómo escala:* mejorable mediante upgrades dedicados.  
*Qué pasa si colapsa:* d permanece, pero su impacto se vuelve irrelevante.

**so — Especialización de Oficio**  
Buff estructural aplicado exclusivamente a d.  
*Qué mide:* profundización del conocimiento técnico.  
*Cómo escala:* mejoras discretas que multiplican d.  
*Qué pasa si colapsa:* el trabajo manual pierde competitividad frente a redes de intercambio.

**e — Trueque**  
Productor alternativo basado en redes de intercambio.  
*Qué mide:* valor generado por conexiones, no por esfuerzo individual.  
*Cómo escala:* lineal por unidad, pero su impacto depende de me.  
*Qué pasa si colapsa:* el sistema pierde capacidad de generar valor mediante relaciones.

**me — Red de Intercambio**  
Multiplicador del sistema de trueque.  
*Qué mide:* densidad y calidad de las conexiones económicas.  
*Cómo escala:* mejoras dedicadas que amplifican e.  
*Qué pasa si colapsa:* el trueque existe, pero su potencial permanece latente.

### 2.4 Dominancia del sistema

En cada momento, el sistema está dominado por uno de tres regímenes:

- **Régimen CLICK:** Δ$_click > Δ$_pasivo  
  El sistema depende de intervención activa. Es joven o está en crisis.

- **Régimen TRABAJO MANUAL:** d · md · so > e · me  
  El sistema privilegia la producción individual organizada.

- **Régimen TRUEQUE:** e · me > d · md · so  
  El sistema ha evolucionado hacia economía de red.

*Qué mide la dominancia:* el estado evolutivo del sistema.  
*Qué pasa si no hay dominancia clara:* inestabilidad transitoria. El sistema está mutando.

## 3. Cognición (μ)

### 3.1 Definición de capital cognitivo

El capital cognitivo no es dinero. No es producción directa.  
Es la capacidad del sistema para **organizar, transmitir y estabilizar** mejoras.

Se define como:

```
μ(n) = 1 + log(1 + n) · β
```

Donde:
- **n** = nivel cognitivo (cognitive_level)
- **β** = coeficiente de impacto cognitivo (actualmente 0.08, mejorable)
- **μ ≥ 1** siempre

### 3.2 Propiedades biológicas de μ

**Crecimiento logarítmico:**  
μ crece rápido al principio, luego se desacelera. Esto no es un defecto: es realismo.  
El conocimiento temprano tiene alto impacto. El conocimiento avanzado produce mejoras marginales.

**Rendimientos decrecientes:**  
Cada unidad de n produce menos Δμ que la anterior.  
*Por qué:* aprender a sumar es más transformador que optimizar algoritmos cuánticos.

**Nunca se aplana completamente:**  
A diferencia de funciones con asíntotas, μ siempre crece. Lento, pero crece.  
*Interpretación:* no hay techo absoluto para la cognición.

### 3.3 β como parámetro jugable

Actualmente β = 0.08 es fijo.  
Pero β puede evolucionar:

```
μ(n) = 1 + log(1 + n) · β(upgrades)
```

Esto habilita meta-productores que no generan dinero, sino **calidad cognitiva**.

**Ejemplos de mejoras a β:**
- Educación formal
- Infraestructura de investigación
- Cultura técnica
- Instituciones de memoria colectiva

*Qué mide β:* la calidad de las instituciones que sostienen el conocimiento.  
*Cómo escala β:* mediante inversiones costosas y lentas.  
*Qué pasa si β colapsa:* μ deja de tener impacto real. El sistema acumula conocimiento inerte.

### 3.4 Por qué μ no es daño

μ no es fricción.  
μ no es costo.  
μ es **estructura cognitiva activa**.

Confundir μ con daño es como confundir el cerebro con una herida.

## 4. Esqueleto Estructural (cₙ, κμ, fⁿ)

### 4.1 Anatomía de la persistencia

El sistema tiene tres capas de estructura:

**c₀ — Persistencia base**  
El mínimo estructural del sistema. No se puede reducir.  
*Qué mide:* el "piso" por debajo del cual el sistema colapsa completamente.

**k — Constante de persistencia**  
Factor estructural fijo que multiplica el impacto de las mejoras.  
*Qué mide:* cuánto persiste cada mejora estructural en el tiempo.

**n — Upgrades estructurales**  
Cantidad total de mejoras estructurales desbloqueadas.  
*Qué mide:* complejidad acumulada del sistema.

### 4.2 κμ — Persistencia efectiva cognitiva

μ no actúa directamente sobre cₙ.  
μ **modula k**:

```
κμ = k · (1 + α · (μ - 1))
```

Donde:
- **α** = coeficiente de impacto cognitivo (actualmente 0.55)

*Por qué existe κμ:*  
Sin μ, las mejoras estructurales serían lineales y predecibles.  
Con μ, cada mejora se amplifica proporcionalmente a la capacidad cognitiva del sistema.

*Interpretación biológica:*  
κμ es la **memoria estructural reforzada por cognición**.  
Un sistema con alto μ no solo recuerda mejor: aplica mejor lo que recuerda.

*Qué mide κμ:* cuánto de cada mejora estructural persiste de manera efectiva.  
*Cómo escala:* crece con μ, pero con rendimientos decrecientes (por el logaritmo interno de μ).  
*Qué pasa si colapsa:* las mejoras estructurales se vuelven temporales. El sistema olvida.

### 4.3 fⁿ — Persistencia teórica esperada

El sistema tiene un estado estructural ideal hacia el cual intenta converger:

```
fⁿ = c₀ · κμ^(1 - 1/n)
```

Donde:
- **n** = cantidad de upgrades estructurales

*Qué mide fⁿ:* el estado estructural que el sistema "debería" tener según su historia de mejoras.  
*Por qué no es instantáneo:* porque la estructura real (cₙ) no puede saltar. Debe converger.

*Interpretación biológica:*  
fⁿ es el **genotipo estructural**.  
cₙ es el **fenotipo estructural** (lo observado en tiempo real).

### 4.4 cₙ — Persistencia dinámica observada

cₙ no es un valor fijo. Es un estado que evoluciona hacia fⁿ mediante convergencia sigmoidal:

```
cₙ(t+Δt) = lerp(cₙ, fⁿ, α(n)·Δt)
```

*Qué mide cₙ:* el estado estructural real del sistema en el momento t.  
*Cómo escala:* converge hacia fⁿ a una velocidad controlada por α(n).  
*Qué pasa si colapsa:* el sistema pierde toda amplificación estructural. Vuelve a c₀.

## 5. Convergencia y Sigmoide

### 5.1 La función α(n)

La velocidad de convergencia no es constante. Está regulada por una sigmoide:

```
α(n) = 1 / (1 + e^(-0.35 · (n - 6)))
```

*Por qué existe α(n):*  
Sin ella, cₙ saltaría instantáneamente a fⁿ cada vez que se compra una mejora.  
Con ella, el sistema tiene **inercia estructural**.

### 5.2 Fases de convergencia

**Fase temprana (n < 6):**  
α(n) es bajo. El sistema converge lentamente.  
*Interpretación:* las primeras mejoras son difíciles de integrar. El sistema aún no tiene infraestructura para adoptarlas rápido.

**Fase de transición (n ≈ 6):**  
α(n) crece rápidamente. La convergencia se acelera.  
*Interpretación:* el sistema ha desarrollado capacidad de absorber cambios estructurales.

**Fase madura (n > 10):**  
α(n) se estabiliza cerca de 1. La convergencia es casi instantánea.  
*Interpretación:* el sistema ha alcanzado madurez estructural. Nuevas mejoras se integran sin fricción.

### 5.3 Parámetros jugables de la sigmoide

La sigmoide tiene dos parámetros:

**0.35 — Pendiente**  
Controla qué tan abrupta es la transición entre fases.  
*Qué pasa si aumenta:* la transición es más violenta. Crisis más abruptas.  
*Qué pasa si disminuye:* la transición es más suave. Cambios graduales.

**6 — Centro**  
Controla en qué momento ocurre la transición.  
*Qué pasa si aumenta:* la madurez estructural llega más tarde. Fase temprana más larga.  
*Qué pasa si disminuye:* el sistema madura más rápido. Menos tiempo en fase de aprendizaje.

### 5.4 Futuro: sigmoide como vector de crisis

Actualmente la sigmoide es fija.  
Pero podría volverse un parámetro afectado por:

- Regulación (suaviza la pendiente)
- Instituciones (desplaza el centro hacia la izquierda)
- Crisis (desplaza el centro hacia la derecha)
- Shocks externos (altera temporalmente α)

## 6. Estrés (ε)

### 6.1 Dos formas de estrés

El sistema tiene dos medidas de distancia estructural:

**ε_modelo — Distancia estructural teórica**

```
ε_modelo = |fⁿ - cₙ(modelo)|
```

*Qué mide:* cuán lejos está el sistema de su estado estructural ideal.  
*Estado actual:* solo diagnóstico. No tiene consecuencias jugables.

**ε_runtime — Estrés estructural del sistema económico**

```
dε/dt = impacto de producción, complejidad, velocidad de crecimiento
```

*Qué mide:* tensión acumulada por el sistema al operar.  
*Estado actual:* integrado en el DLC Fúngico. Produce nutrientes.

### 6.2 Interpretación biológica de ε_runtime

ε_runtime no es "daño".  
Es **entropía económica**.

Todo sistema que produce genera desorden:
- fricciones no capturadas
- ineficiencias latentes
- complejidad no metabolizada
- estrés estructural acumulado

Sin un mecanismo que absorba ε_runtime, el sistema colapsa.  
Ese mecanismo es la biosfera.

### 6.3 Estados de estrés

**Bajo estrés (ε < 0.1):**  
El sistema opera dentro de sus límites estructurales.  
*Síntoma:* cₙ ≈ fⁿ. Convergencia fluida.

**Estrés moderado (0.1 < ε < 0.3):**  
El sistema comienza a acumular tensión.  
*Síntoma:* cₙ se retrasa respecto a fⁿ. Aparece fricción.

**Estrés alto (ε > 0.3):**  
El sistema está operando más allá de su capacidad estructural.  
*Síntoma:* cₙ diverge de fⁿ. Sin biosfera, esto es pre-colapso.

### 6.4 Futuro: ε como vector de fricción

Propuesta de consecuencias jugables de ε_runtime:

```
Δ$_efectivo = Δ$ · (1 - ε_runtime)
```

O más sofisticado:

```
Δ$_efectivo = Δ$ / (1 + ε_runtime)
```

Esto introduce:
- retroalimentación negativa natural
- límites al crecimiento explosivo
- necesidad de inversión en biosfera

## 7. Biosfera Fúngica

### 7.1 La segunda ley termodinámica económica

Antes del DLC Fúngico, el sistema tenía:
- producción
- estructura
- eficiencia
- fricción (ε)

Pero no tenía:
**un sistema que absorba entropía**.

Eso es la biosfera.

### 7.2 Ciclo termodinámico completo

```
Economía → ε → Nutrientes → Biomasa → ε_efectivo
```

Formalmente:

**Generación de nutrientes:**

```
dN/dt = kₙ · ε_runtime
```

Donde:
- **N** = nutrientes (entropía económica capturada)
- **kₙ** = tasa de conversión de estrés a nutrientes

*Qué mide:* velocidad de captura de la entropía económica.  
*Interpretación biológica:* el estrés no desaparece. Se convierte en sustrato metabolizable.

**Crecimiento de biomasa:**

```
dB/dt = kᵦ · N · μ
```

Donde:
- **B** = biomasa (capacidad del sistema para absorber estrés)
- **kᵦ** = tasa de conversión de nutrientes a biomasa
- **μ** = capital cognitivo

*Por qué depende de μ:*  
Sin cognición, los nutrientes no se organizan.  
μ es el "cerebro" de la biosfera: decide cómo distribuir recursos.

**Consumo de nutrientes:**

```
N ← N - dB/dt
```

Los nutrientes se agotan al generar biomasa.  
*Interpretación:* la biosfera metaboliza el estrés. No lo acumula indefinidamente.

### 7.3 Estados de la biosfera

**Biosfera inexistente (B = 0):**  
ε_runtime crece sin control.  
*Síntoma:* crisis inevitable. El sistema no puede digerir su propio crecimiento.

**Biosfera emergente (0 < B < 10):**  
ε_runtime se reduce parcialmente.  
*Síntoma:* el sistema comienza a estabilizarse, pero sigue frágil.

**Biosfera madura (B > 50):**  
ε_runtime se mantiene bajo incluso con alta producción.  
*Síntoma:* el sistema puede crecer sin colapsar.

## 8. Biomasa

### 8.1 Biomasa como amortiguador de crisis

La biomasa no elimina el estrés.  
Lo vuelve **digerible**.

```
ε_efectivo = ε / (1 + B)
```

*Qué mide:* cuánto del estrés original persiste después de ser metabolizado por la biosfera.  
*Cómo escala:* rendimientos decrecientes. Cada unidad de biomasa reduce menos ε que la anterior.  
*Qué pasa si colapsa:* ε_efectivo = ε. El sistema pierde toda capacidad de amortiguar crisis.

### 8.2 Biomasa como booster económico

La biomasa no aumenta dinero directamente.  
Modula eficiencia.

Definimos:

```
β(B) = 1 + log(1 + B)
```

Y afecta múltiples variables:

| Variable | Efecto |
|----------|--------|
| md | md · β(B) |
| me | me · β(B) |
| so | so · (1 + 0.5β) |
| μ | μ · (1 + 0.3β) |

*Interpretación biológica:*  
La biosfera no produce. **Cataliza**.  
Es el sistema circulatorio de la economía: no genera energía, pero la distribuye mejor.

### 8.3 Por qué β(B) es logarítmico

β(B) crece rápido al principio, luego se desacelera.  
*Por qué:* una biosfera pequeña tiene alto impacto marginal.  
Una biosfera madura produce mejoras incrementales.

Esto previene:
- explosiones descontroladas
- loops infinitos de biomasa → producción → más biomasa
- colapso del balance económico

### 8.4 Estados de amplificación

**Sin amplificación (B = 0):**  
β(B) = 1. No hay efecto.

**Amplificación temprana (0 < B < 20):**  
β(B) crece rápido. Alto impacto marginal.

**Amplificación madura (B > 100):**  
β(B) crece lento. Mejoras incrementales.

## 9. Micelio

### 9.1 Definición de micelio

El micelio es **biomasa cristalizada**.  
Es memoria biológica permanente.

```
M = √B
```

*Propiedades:*
- Nunca decrece
- Afecta estructura
- No depende de nutrientes

### 9.2 Micelio como estructura cognitiva

El micelio amplifica μ:

```
μ_efectivo = μ · (1 + 0.1M)
```

*Qué mide:* capacidad cognitiva reforzada por memoria biológica.  
*Interpretación:* el micelio es la "biblioteca" del sistema. No piensa, pero permite que μ piense mejor.

### 9.3 Micelio como rigidez productiva

El micelio refuerza κμ:

```
κμ = k · (1 + α(μ_efectivo - 1))
```

*Efecto en cadena:*  
M → μ_efectivo → κμ → fⁿ → cₙ

*Interpretación:*  
El micelio no produce directamente.  
Refuerza la capacidad del sistema para mantener estructura bajo estrés.

### 9.4 Por qué M = √B

La raíz cuadrada tiene dos propiedades clave:

**Crece más lento que B:**  
Evita que el micelio domine el sistema.

**Nunca decrece:**  
Incluso si B cae, M persiste.

*Interpretación biológica:*  
El micelio es **memoria estructural irreversible**.  
La biosfera puede colapsar. El micelio no.

### 9.5 Estados de micelio

**Sin micelio (M = 0):**  
El sistema no tiene memoria biológica.  
*Síntoma:* cada crisis es como la primera. No hay aprendizaje.

**Micelio emergente (0 < M < 5):**  
El sistema comienza a desarrollar memoria.  
*Síntoma:* las crisis se vuelven menos destructivas.

**Micelio maduro (M > 15):**  
El sistema tiene memoria estructural profunda.  
*Síntoma:* crisis locales no afectan el núcleo estructural.

## 10. Metabolismo

### 10.1 Definición de metabolismo

El metabolismo no es una variable del sistema.  
Es un **indicador de salud**.

```
Metabolismo = B / Δ$
```

*Qué mide:* cuánta biosfera existe por cada unidad de energía producida.  
*Interpretación:* ratio de amortiguación del sistema.

### 10.2 Estados metabólicos

| Metabolismo | Estado | Síntoma |
|-------------|--------|---------|
| > 0.12 | Estable | El sistema puede absorber shocks sin colapsar |
| 0.06–0.12 | Forzado | El sistema opera cerca de su límite estructural |
| 0.03–0.06 | Agotado | La biosfera no puede metabolizar todo el estrés |
| < 0.03 | Crítico | Pre-colapso. Crisis inevitable |

### 10.3 Por qué el metabolismo no es un evento

El metabolismo no es:
- un trigger
- un threshold binario
- una mecánica puntual

El metabolismo es:
- una **condición física continua**
- un **diagnóstico en tiempo real**
- una **medida de fragilidad**

### 10.4 Metabolismo bajo como señal

Cuando el metabolismo cae por debajo de 0.06:

**El sistema está diciendo:**  
"Estoy creciendo más rápido de lo que mi biosfera puede metabolizar."

**Opciones del jugador:**
1. Frenar producción (reducir Δ$)
2. Invertir en biosfera (aumentar B)
3. Aceptar crisis estructural (dejar que ε explote)

No hay opción correcta universal.  
Cada decisión tiene consecuencias.

## 11. Estados del Sistema

### 11.1 Matriz de estados

El sistema puede estar en uno de varios estados según la combinación de variables:

| Estado | Δ$ | B | M | Metabolismo | Síntoma |
|--------|-----|---|---|-------------|---------|
| **Primitivo** | Bajo | 0 | 0 | N/A | Sin biosfera. Sin crisis. Sin complejidad. |
| **Expansión saludable** | Alto | Alto | Medio | > 0.12 | Crecimiento sostenible. |
| **Sobrecalentamiento** | Muy alto | Medio | Bajo | < 0.06 | Crecimiento más rápido que metabolismo. |
| **Post-colapso** | Bajo | Bajo | Alto | > 0.15 | El micelio persiste. Recuperación lenta. |
| **Madurez estructural** | Alto | Muy alto | Alto | > 0.15 | Sistema estable con memoria profunda. |

### 11.2 Transiciones de estado

**Primitivo → Expansión saludable:**  
Inversión temprana en biosfera durante el crecimiento inicial.

**Expansión saludable → Sobrecalentamiento:**  
Producción crece más rápido que la biosfera puede absorber.

**Sobrecalentamiento → Post-colapso:**  
ε_runtime explota. La producción se derrumba. Solo el micelio persiste.

**Post-colapso → Madurez estructural:**  
Reconstrucción apoyada en la memoria estructural del micelio.

### 11.3 Loops positivos y negativos

**Loop positivo de biomasa:**

```
Δ$ ↑ → ε ↑ → N ↑ → B ↑ → β(B) ↑ → Δ$ ↑
```

*Problema:* sin límites, este loop explota.  
*Solución:* β(B) es logarítmico. Los rendimientos decrecen.

**Loop negativo de crisis:**

```
Δ$ ↑ → ε ↑ → crisis → Δ$ ↓ → B ↓ → ε_efectivo ↑ → crisis más profunda
```

*Problema:* sin micelio, este loop destruye el sistema.  
*Solución:* M persiste incluso cuando B colapsa. Hay piso estructural.

## 12. Crisis y Expansión Futura

### 12.1 Capas previstas no implementadas

El sistema está diseñado para evolucionar hacia:

**Meta-productores:**  
No producen dinero. Producen parámetros.

| Meta-productor | Afecta |
|----------------|--------|
| Educación | β (impacto de μ) |
| Instituciones | ε_runtime |
| Regulación | Suaviza sigmoide |
| Cultura técnica | α de κμ |

**Regulación:**  
Limita extremos. Reduce volatilidad.  
Costo: menor crecimiento explosivo.  
Beneficio: menor riesgo de colapso.

**Instituciones:**  
Estabilizan cₙ frente a shocks.  
Reducen costos de upgrades estructurales.  
Permiten runs más largas.

**Entropía/Deuda/Complejidad:**  
Capa negativa que crece con:
- structural_upgrades
- velocidad de crecimiento
- ε_runtime sostenido

Obliga a invertir en regulación e instituciones.

### 12.2 ε_runtime como vector de fricción

Propuesta futura:

```
Δ$_efectivo = Δ$ / (1 + ε_runtime)
```

O con saturación:

```
Δ$_efectivo = Δ$ · (1 - tanh(ε_runtime))
```

*Efecto:* el sistema no puede crecer indefinidamente sin consecuencias.  
*Gameplay:* obliga a balancear producción y biosfera.
